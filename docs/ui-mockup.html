i.

Home is a roster of threads

The feed becomes a chats list. Each row is a reopenable conversation, bucketed by machine. The flagged thread at the top wants a decision; everything else stays calm.

  • Fleet strip shows paired machines — tap to filter, last card is always + pair machine.
  • Section headers group threads by machine and offer an inline + new chat link.
  • FAB is the global new-chat entry — opens the composer sheet.
  • Tabbar exposes machines and repos as first-class destinations.
← UserDO.snapshot drives the fleet strip · each row keyed by {machine_id, project_id, branch} — same key the supervisor publishes.
ii.

Open a thread, edit the conversation

Inside a chat, dispatches stack as bubbles. Your redirects sit on the right. Decision strips render inline where the agent paused for permission, and tool calls collapse into muted dashed rows so the prose is what reads.

  • Three voices — agent dispatch (white), peer ↔ message (cream), tool call (dashed).
  • Committing bubbles get the oxblood border + countdown; tap to redirect before it commits.
  • Composer always available — typing here sends a push_event redirect, even mid-task.
  • Aux chips above the field offer one-tap redirects (stop, summarize, follow).
← agent bubble = report_progress { note } · me bubble = push_event { event_kind:'redirect' } · decision = permission_request ↑ permission_verdict

iii.

Compose a chat in one sheet

The FAB lifts a single sheet that asks the three things needed to spawn a supervisor: which machine, which repo, which branch. Each picker ends with an explicit add tile so onboarding gaps surface here, not behind a settings menu.

  • Machine — paired Bun-CLI hosts, with + pair machine as the rightmost tile.
  • Repo — projects registered on the chosen machine; + add repo opens the registration flow scoped to that machine.
  • Branch — defaults to a new branch from main; existing branches available, busy ones marked.
  • Prompt — the first message; tap start spawns the worktree, attaches Claude, opens the thread.
↑ UI sends create_session { machine_id, project_id, branch, prompt } · UserDO routes to MachineDO · supervisor materializes worktree, launches Claude, returns session_registered.
iv.

Pair a machine with a 6-digit code

Adding a machine means running the Bun CLI somewhere and binding it to your account. The screen shows the install command, a one-shot pairing PIN, and a live indicator that flips when the supervisor connects.

  • Install — copy-pasteable line for any shell; the same binary on every OS.
  • Pair — six-digit PIN with a TTL; ties the supervisor's first WS handshake to your UserDO.
  • Heartbeat — once connected, the row goes green and the sheet auto-dismisses.
↑ POST /pair on UserDO mints PIN · ↓ supervisor sends pair { pin } on first WS · UserDO writes { machine_id, user_id } and accepts subsequent heartbeat frames.

v.

Repos: pick or paste

The supervisor scans configured roots and surfaces git repos it can see. Already-registered ones are checked. Tap + add to register, or copy the equivalent CLI command if you'd rather do it from a shell.

  • Detected list — green check for registered, accent + add for the rest.
  • Shell parity — every UI action has its CLI twin (setu project add) shown right below. The UI is a façade over the same store.
  • Bring-your-own — for repos not yet cloned, the screen shows the two-line flow rather than pretending to handle git auth.
  • Per-repo wiring — .setu/channels.json is the per-repo MCP config; surfaced as a footnote, not pushed.
↑ project_add { project_id, root_path } on the supervisor's WS · MachineDO updates its registry · UserDO snapshot regenerates → home repos tab refreshes live.

Net-new for chat-pane UX

create create_session reply tool — UI → UserDO → MachineDO → supervisor spawn (worktree + Claude). Replaces "ssh and run setu manually".

pair POST /pair mints PIN; supervisor's first WS frame is pair { pin } — binds machine_id ↔ user_id.

repo project_add already exists on bun-cli; UI now drives it via the supervisor channel.

commit dispatch { kind:'committing', default_after_ms } renders the oxblood countdown bubble.

redirect chat composer = persistent push_event stream into the active session, not just paired with a request_input.

Out of scope

raw Raw Claude assistant turns / thinking — still not on the wire. Thread shows authored prose + tool-call timeline. Capture via stdio is a separate workstream.

clone The UI does not run git clone — auth is the user's. We surface the command, not the secret.

pwd No file picker into arbitrary paths from the phone — the supervisor scans configured roots; everything else is a CLI command.

setu is a bridge. The phone is the editor's desk; the machines do the work.
